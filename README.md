# MklinkTool

一个基于 Java 的命令行工具，用于在 Windows 系统中批量创建目录联接点（Junction）和文件符号链接。支持通配符展开、重名处理、多种替换策略。

## 环境要求

- **操作系统**：Windows（依赖 `mklink` 命令）
- **JDK**：Java 21 或更高版本

## 运行方式

1. 在 https://github.com/THXY-wx/MklinkTool/releases 中下载最新版本的 `MklinkTool.jar` 文件。
2. 在 jar 文件所在目录中以管理员身份打开命令行窗口。
3. 执行以下命令：

```bash
java -jar MklinkTool.jar
```

## 使用方法

工具启动后，按提示交互式输入：

1. **输入原始路径**  
   支持多个路径，每行一个。可用双引号包裹，支持通配符 `*`：
   - `C:\Data\*.txt` – 匹配所有 `.txt` 文件
   - `C:\Projects\*\bin\*` – 递归匹配所有 `bin` 目录下的所有文件
   - 直接输入文件或目录路径（不带通配符）
   - 空行结束输入

2. **输入目标目录**  
   链接将创建在该目录下，名称与原文件/目录同名。

程序自动完成链接创建。

## 链接类型说明

| 原始类型 | 使用的命令 | 链接类型 | 备注 |
|---------|-----------|---------|------|
| **目录** | `mklink /J` | 目录联接点（Junction） | 支持跨卷，兼容性好 |
| **文件** | `mklink`（不带选项） | 文件符号链接（Symbolic Link） | 支持跨卷，但需要管理员权限 |

### 关于目录联接点（Junction）
- 仅适用于目录，透明访问，删除不影响原始数据。
- 支持跨卷，不支持网络路径。
- 大多数 Windows 工具（robocopy、xcopy 等）能正确处理。

### 关于文件符号链接
- 适用于单个文件，访问链接等同于访问原始文件。
- 支持跨卷，可指向不存在的目标（悬垂链接）。
- **需要管理员权限**才能创建（或启用开发者模式），否则可能会失败。

## 注意事项

- 仅支持 Windows 系统（非 Windows 会自动退出）。
- 文件符号链接需要以管理员身份运行程序，否则可能会失败。
- 删除已有链接时仅移除链接本身，不会影响原始数据。
- 通配符 `*` 可出现在路径任意位置，支持多级展开。
